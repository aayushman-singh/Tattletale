# Session Files

Tattletale's scrapers authenticate against each social platform via session files. These hold live login state — **never commit them**.

## Two kinds of session files

### 1. Telethon `.session` / `.session-journal`

For Telegram. Generated by Telethon (Python).

```bash
python scraper/telegram_login.py --phone +YOUR_NUMBER
```

Creates `session_+YOUR_NUMBER.session` (SQLite) and `.session-journal` (write-ahead log) in the repo root. Anyone with these files can:

- Read your DMs
- Send messages as you
- See group memberships and contacts

### 2. Playwright/Puppeteer cookie-jar `session.json`

For Instagram, Facebook, X/Twitter, Mastodon, Discord, YouTube, Google Drive — anything browser-driven. Generated by the headed scraper login flow.

```bash
cd scraper && npm run login -- --platform instagram
```

Creates `session.json` with `cookies` + `origins.localStorage` arrays. Anyone with the `sessionid` cookie can log in as you on Instagram. Same risk model on every other platform.

## Naming

| File | Purpose |
| --- | --- |
| `session_+<phone>.session` | Telethon, phone-keyed |
| `session_name.session` | Telethon, label-keyed |
| `session.json` | Playwright cookie jar (per platform run) |
| `scraper/session.json` | Same, run from scraper directory |
| `*.session-journal` | SQLite WAL for Telethon |

## If a session leaks

1. Open the affected app → log out **all** sessions:
   - Telegram: Settings → Privacy and Security → Active Sessions → Terminate All Other Sessions
   - Instagram: Settings → Security → Where you're logged in → log out each
   - Facebook: Settings → Security and Login → Where you're logged in → Log out of all sessions
   - X/Twitter: Settings → Security → Sessions → Log out of all other sessions
   - Google: `myaccount.google.com/security` → Your devices → Sign out
2. Delete the leaked file from local disk.
3. If pushed to a remote repo, purge from git history with [`git-filter-repo`](https://github.com/newren/git-filter-repo):

```bash
pip install git-filter-repo
git filter-repo --invert-paths --path session.json --path "session_+YOUR_NUMBER.session" --force
git push --force origin main
```

## For learners cloning this repo

You will not find any working `.session` or live cookie `session.json` in this repo. Every committed example uses `YOUR_*` placeholders. Generate your own by running the platform's login flow before scraping.
